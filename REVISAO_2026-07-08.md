## 🔴 CRITICAL
1. **Build Linux quebrado** — `IO.Epoll.pas:37` e `IO.IOUring.pas:69`: `FShutdown: Integer` lido com `TInterlocked.Read` (só há overload `Int64`) → E2033, não compila em Linux64 (comprovado dcc64). IOCP/RIO já usam Int64. **Fix: mudar os 2 campos p/ Int64.**
2. **Overflow HPACK** — `HTTP2.HPACK.pas:590`: `Integer(LLen) > ABufLen` com LLen ≥ $80000000 vira negativo → passa bounds-check → SetLength+Move ~2GB de buffer de KB → OOB read/crash remoto. Fix: comparar unsigned + detectar overflow em `_HpackDecodeInt`.

## 🟠 HIGH
### HTTP/2 (DoS por cliente único)
- `HTTP2.pas:982` corpo de request sem teto; flow-control auto-reabastece → OOM.
- `HTTP2.HPACK.pas:728` HPACK bomb — sem MAX_HEADER_LIST_SIZE; refs indexadas expandem sem limite + array O(n²).
- `HTTP2.pas:869` MAX_CONCURRENT_STREAMS não imposto + sem defesa rapid-reset (CVE-2023-44487).
### Ciclo de vida / shutdown (crash)
- `HttpServer.pas:1098` Stop() desmonta worker pool com trabalho em voo (drain = evento manual-reset disparo único) → AV/UAF; pior em GracefulReload.
- `HttpServer.pas:1102` teardown libera SSL ANTES de drenar pool → UAF do handle SSL (TLS + handler lento).
- `HttpServer.pas:866` SetSyncDispatch sem guarda FActive → FreeAndNil(FDispatcher) sob tráfego = UAF.
- `HttpServer.pas:1167` vazamento de socket no except de _OnNewSocket (falta SocketClose) → exaustão fd.
### WebSocket / Segurança
- `WebSocket.Manager.pas` fragmentação (continuation) não tratada → mensagens fragmentadas corrompidas silenciosamente.
- `WebSocket.pas` bomba de deflate: Decompress infla sem teto antes do limite → OOM.
- `Middleware.RateLimit.pas:58` chave = X-Forwarded-For sem gate de proxy confiável → bypass total do rate limit + spoof de cota.

## 🟡 MEDIUM
- H1 off-by-one: `GET / HTTP/1.0\r\n\r\n` sem headers → 400 espúrio (`HTTP1.Parser.pas:488` e `:739`).
- H2: frame-size validado contra valor do peer e tarde; END_STREAM perdido em CONTINUATION; stream-id do cliente não validado (par colide c/ push → crash); dyn-table size do decoder ditado pelo cliente; janela inicial 0 → resposta bufferizada indefinidamente.
- Segurança: Content-Type não sanitizado → response splitting (`ResponseBuilder.pas:262`); Proxy Protocol sem allowlist → IP spoofing; `IsIPInCIDR` fail-open em octeto >255 (`Security.pas:145`).
- Concorrência: `Pool.Workers.Shutdown` descarta itens enfileirados → vaza AddRef + FInFlightCount.
- Server: pool estático TBufferPool misturado com IBufferPool injetado → contabilidade quebrada c/ pool custom.

## ⚪ LOW (principais)
WS: controle >125/FIN=0 aceito, não-mascarado aceito, sem UTF-8, opcodes reservados ignorados, handshake não valida Version:13. H1: cap chunk-ext rejeita extensões válidas; Router param case-insensitive vs estáticas case-sensitive; keep-alive HTTP/1.0 divergente no parser Lightweight. H2: Huffman leniente, pseudo-headers não validados (risco smuggling h2→h1). Buffer pool: TLC das IO-threads vaza no reload. GracefulReload: SIGTERM ignorado fora de console. MemoryManager.Linux: ponteiros de MM = nil (AV potencial).

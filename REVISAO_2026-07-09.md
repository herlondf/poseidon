## 🔴 CRITICAL

### C1. ProxyProtocol fail-close por default quebra o feature (regressão introduzida pelo fix M3)
`src/Poseidon.Net.Dispatcher.pas:245-248` — `StepProxyProtocol` chama `TryParseProxyProtocolAuto` sem passar `APeerAddr`/`ATrustedProxies`. Defaults (`''`/`nil`) resultam em fail-close → PP header nunca parseado. Usuário configurando `ProxyProtocol := ppV2` behind HAProxy vê server ignorar completamente o PP header; `RemoteAddr` retém IP do load balancer para toda requisição. **Rate limiter, IP allowlist e audit log ficam broken.** Fix: adicionar `FTrustedProxies: TArray<string>` em `TPoseidonNativeServer` + propagar via `TDispatchConfig` até `StepProxyProtocol`.

### C2. IdleSweep faz `shutdown()` de socket handle recyclado — mata conexão de outro cliente
`src/Poseidon.Net.IdleSweep.pas:100-118` — `Snapshot` faz `AddRef` no `TNativeConn`, mas o campo `Socket` continua stale após `SocketClose(LConn)` (IOCP `AddRecycled` reusa handle imediatamente; kernel Linux faz FD reuse). Sequência: T-Sweep captura Snapshot → T-IO fecha ConnA + recicla socket X → T-Accept aceita ConnB e reusa socket X → T-Sweep executa `shutdown(X, SD_BOTH)` matando conexão do cliente B. **Prova IOCP: TSocketPool.AddRecycled reusa handles em AcceptEx.** Fix: `SocketClose` seta `LConn.Socket := INVALID_SOCKET` após close syscall; `ShutdownConn` checa sentinela antes de shutdown.

## 🟠 HIGH

### H1. Dispatcher error paths (400/413) DoS loop em conexão keep-alive
`src/Poseidon.Net.Dispatcher.pas:285-293 / 355-365 / 396-406` — `StepSizeCheck`, `StepParseHTTP1Full`, `StepParseHTTP1Lightweight` emitem `Connection: close` no header MAS não setam `LConn.KeepAlive := False` nem consomem `AccumBuf`. `OnSendComplete` (HttpServer.pas:529) vê KeepAlive=True + AccumLen>0 → `_ProcessRecv(AConn, nil, 0)` → `_DispatchAccumBuf` → mesmo 400 outra vez. Loop até cliente desistir. Fix: nos três branches, antes de enviar erro, `LConn.KeepAlive := False` e `LConn.AccumLen := 0`.

### H2. SETTINGS INITIAL_WINDOW_SIZE delta overflow silencioso (RFC 7540 §6.9.2)
`src/Poseidon.Net.HTTP2.pas:815-827` — `Inc(SendWindow, LDelta)` sem checar overflow. Peer com `SendWindow=$7FFFFFFF` + SETTINGS INITIAL_WINDOW_SIZE=$7FFFFFFF → LDelta positivo → SendWindow vira negativo signed Integer. RFC exige connection error `FLOW_CONTROL_ERROR`. Fix: `if (LDelta > 0) and (LStream.SendWindow > MaxInt - LDelta) then GOAWAY(FLOW_CONTROL_ERROR); else Inc(...)`.

### H3. `_HandleWindowUpdate` connection-level dreena 1 stream + `Break` — outros stalled
`src/Poseidon.Net.HTTP2.pas:943-949` — Se N streams têm PendingBody bloqueados por conn-window exausto, apenas o primeiro drena. Peer não tem razão para mandar novo WINDOW_UPDATE pois já concedeu crédito. Stall permanente. Fix: iterar todos streams com pending até conn-window esgotar de novo.

### H4. UAF em SSLHandle quando pool worker excede FDrainTimeoutMs
`src/Poseidon.Net.HttpServer.pas:1131-1161` — `FRequestPool.Shutdown` faz Break sem join (`Pool.Workers.pas:306`), `JoinWorkers` só sincroniza IO workers, teardown libera `SSLHandle` de stragglers. Pool worker que retorna do handler stuck faz `SSLWrite(LConn.SSLHandle, ...)` no ponteiro `SSL_free`d. Fix: FRequestPool.Shutdown deve join workers ou emitir sentinela para eles NÃO tocarem SSLHandle após timeout.

### H5. WebSocket `TWebSocketManager.DropConnection` **nunca é chamado** — leak FragStates + pointer reuse
`src/Poseidon.Net.WebSocket.Manager.pas:63,147` — Grep global mostra 0 call-sites. `TPoseidonNativeServer._CloseConn` (HttpServer.pas:1219-1245) libera conn sem chamar `FWSManager.DropConnection`. FFragStates cresce indefinidamente sob conexões WS abortadas mid-fragment. Fix: `FWSManager.DropConnection(AConn)` em `_CloseConn` antes de `Release`.

### H6. Digest middleware sem replay protection + nonce fraco
`middlewares/Poseidon.Middleware.Digest.pas:61-68` — `GenerateNonce` usa `Random(MaxInt)` (não seeded, previsível), zero nonce store, zero `nc` counter tracking. Header capturado é replayable indefinidamente. + `SameText` compare não constante-tempo. Fix: `TCryptoRandom`, dict de nonces emitidos com expiração, tracking de `nc`, `CompareMem` constante.

### H7. JWT middleware — compare não constante-tempo + `alg` não validado + `nbf`/`iat` ignorados
`middlewares/Poseidon.Middleware.JWT.pas:60,74-96` — `Result := LExpected = ASignature` (short-circuit timing side-channel). Header nunca parseado (`alg` claim não validado — HS/RS confusion latente). `nbf`/`iat` ignorados (token futuro aceito imediatamente). Fix: `CompareMem` tempo constante + validar `alg` header exato + rejeitar se `now < nbf`.

### H8. JSON injection via `E.Message` (Validation/Logger/Proxy)
`middlewares/Poseidon.Middleware.Validation.pas:37`, `Logger.pas:85-89`, `Proxy.pas:86` — concatenação `'"detail":"' + E.Message + '"}'` sem escape. Qualquer `"`, `\`, CR ou LF quebra JSON. Log injection possível via X-Request-ID cliente. Fix: helper `JSONEscape` compartilhado.

### H9. Logger file middleware não thread-safe
`middlewares/Poseidon.Middleware.Logger.pas:93-107` — Cria `TStreamWriter` novo por request com sharing default. 2 workers concorrentes → `EFOpenError` intermitente no Windows; Linux interleaves partial lines. Fix: singleton `TStreamWriter` protegido por lock, ou append via `TFileStream` com share compatível.

### H10. Metrics unbounded cardinality DoS
`middlewares/Poseidon.Middleware.Metrics.pas:180` — Bucket key = `ACtx.Path` raw. Cliente hostil enviando `/x1`, `/x2`, ... `/xN` infla `FBuckets` sem limite → memory exhaustion. Além disso `LPair.Key` embedded raw em exposição Prometheus (linha 117/132) — path com `"` corrompe scrape. Fix: cap `MaxUniquePaths` + escape.

### H11. CircuitBreaker half-open sem limite de probes concorrentes
`middlewares/Poseidon.Middleware.CircuitBreaker.pas:134-135` — `csHalfOpen` retorna True para TODO caller. 100 requests após breaker abrir vêem csHalfOpen sob lock, todos passam, todos batem no backend falho. Fix: `LHalfOpenSlots := 1` counter atomic, exceder = 503.

### H12. Proxy middleware — response headers dropped + XFF pass-through
`middlewares/Poseidon.Middleware.Proxy.pas:73-78,61` — Só status/content-type/body copiados. `Set-Cookie`, `Location`, `Cache-Control`, `WWW-Authenticate` silenciosamente descartados. + X-Forwarded-For do cliente encaminhado verbatim sem anexar IP real (spoof da chain). Fix: copiar todos headers upstream + append peer real ao XFF.

## 🟡 MEDIUM

### M1. HTTP/2 stream-ID reuse não bloqueado (RFC 7540 §5.1.1)
`src/Poseidon.Net.HTTP2.pas:1049-1067` — Stream fechado + removido pode ser reaberto com mesmo ID. Enables trailer-style double dispatch + MAX_CONCURRENT_STREAMS drift. Fix: `if AStreamID <= FLastStreamID and not FStreams.ContainsKey(AStreamID) then PROTOCOL_ERROR`.

### M2. HTTP/1 Compression middleware emite `Content-Encoding: gzip` em 204/304
`middlewares/Poseidon.Middleware.Compression.pas:66-91` — Não consulta `ACtx.Status`. Body dropped pelo builder mas Content-Encoding sai. Cliente que respeita CE tenta inflar zero bytes → falha. Fix: early exit em 204/304/1xx.

### M3. Duplo-encoding quando upstream middleware já setou Content-Encoding
`middlewares/Poseidon.Middleware.Compression.pas:66-91` — Static middleware já emite gzip; Compression roda depois → segundo gzip + segundo header. Fix: iterar ExtraHeaders e sair se `Content-Encoding` já presente.

### M4. Accept-Encoding não parseia `q=0`
`middlewares/Poseidon.Middleware.Compression.pas:77` — `Contains('gzip')` retorna True mesmo para `gzip;q=0` (cliente rejeita). Fix: parser RFC 7231 §5.3.4.

### M5. Missing Vary: Accept-Encoding no caminho não-comprimido
`middlewares/Poseidon.Middleware.Compression.pas:74-91` — Cache middleware pode servir versão errada. Fix: Vary incondicional para recurso elegível.

### M6. Brotli Decompress trata NEEDS_MORE_OUTPUT como sucesso → payload truncado silenciosamente
`src/Poseidon.Net.Brotli.pas:197-224` — `= 0` (erro) tratado; `1` (sucesso) OK; `2` (needs more output) cai como sucesso. Fix: comparar `<> 1` ou realloc e retry.

### M7. mTLS aplicado só ao default SSLCtx — SNI per-host contexts skip verify
`src/Poseidon.Net.SSL.Manager.pas:123-153` — `ConfigureMTLS` toca só `FSSLCtx`. `AddSSLCert` cria novo LCtx sem `SSL_VERIFY_PEER`. Atacante com SNI de host per-host bypassa mTLS. Fix: aplicar mTLS a cada contexto criado.

### M8. Per-host SNI contexts skip SetMinVersion/ALPN
`src/Poseidon.Net.SSL.Manager.pas:130-153` — TLS 1.3 enforcement do default não herda; ALPN callback não wired → HTTP/2 silenciosamente downgrade para HTTP/1.1 em vhosts nomeados.

### M9. Per-IP counter vaza através de Stop → Listen
`src/Poseidon.Net.HttpServer.pas:1143-1161` — Teardown deleta direto de `FConnManager.ConnList` em vez de `Remove`, `FPerIPCount` não decrementa. Após Stop+Listen, `Admit` rejeita legítimos. Fix: usar `FConnManager.Remove(LConn)`.

### M10. ProxyProtocol v1 aceita trailing garbage + no family cross-check
`src/Poseidon.Net.ProxyProtocol.pas:237-247` — `ARemoteAddr := LParts[2]` sem validação. `PROXY TCP4 not-an-ip 10.0.0.1 12345 80\r\n` aceito. Também `PROXY TCP4 ::1 ...` (família mismatch) aceito. Fix: validar `LFamily` vs literal + `IsIPv4(LParts[2])`.

### M11. ProxyProtocol v1 port truncado a 16 bits silenciosamente
`src/Poseidon.Net.ProxyProtocol.pas:244` — `Word(StrToIntDef(LParts[4], 0))` — port 65540 vira 4. Fix: `if LPort > 65535 then reject`.

### M12. StepUpgradeDetection: busca linear O(N) sobre todos headers
`src/Poseidon.Net.Dispatcher.pas:436-446` — 99% dos GETs não são upgrade mas rodam scan completo. Fix: marcar upgrade bit no parser (headers já classificados via LookupHeaderId hash).

### M13. Parser Full aloca `SetLength(AHeaders, 100)` desperdício
`src/Poseidon.Net.HTTP1.Parser.pas:564,650` — 1600 bytes zerados por request. Fix: growth geométrico a partir de 16 ou arena thread-local.

### M14. `_BuildExtraStr` concat O(k) + double pass UTF-16→ASCII
`src/Poseidon.Net.ResponseBuilder.pas:191-208` — 5-7 concats + conversão UTF-16→ASCII de ~200 bytes por response. Fix: escrever direto em `TBytes` da arena.

### M15. `_HTTPDateNow` chama `Now + TZ` a cada request (cache só cobre Format)
`src/Poseidon.Net.ResponseBuilder.pas:174-186` — `TTimeZone.Local.ToUniversalTime(Now)` executa antes de checar cache-por-segundo. Fix: `GetSystemTimeAsFileTime`/`clock_gettime(CLOCK_REALTIME_COARSE)` fora do cache check.

### M16. Validation attributes MinLength/MaxLength medem `ToString`, não string length
`src/Poseidon.Validation.pas:116,134` — Integer 42 (2 chars) reprova em `[MinLength(3)]`. Fix: `if AValue.Kind in [tkUString,tkString,tkLString,tkWString]`.

### M17. RangeAttribute crash em campo não-numérico
`src/Poseidon.Validation.pas:164` — `AValue.AsExtended` levanta EInvalidCast em `[Range] string`. Fix: guard Kind numeric.

### M18. Validate usa `GetFields` — properties silenciosamente ignoradas
`src/Poseidon.Validation.pas:209` — `[Required] property X` não valida. Fix: iterar `GetProperties` também.

### M19. BodyLimit middleware pós-buffer
`middlewares/Poseidon.Middleware.BodyLimit.pas:23` — 1GB body já recebido + copiado antes do 413. Fix: enforcement no parser via Content-Length early reject.

### M20. ProblemDetails leaks raw E.Message em Exception genérica
`middlewares/Poseidon.Middleware.ProblemDetails.pas:51` — SQL/paths internos vazam no `Detail`. Fix: redact em produção; só passthrough para `EPoseidonException`.

### M21. RequestID middleware — client-controlled sem validação
`middlewares/Poseidon.Middleware.RequestID.pas:34-38` — X-Request-ID cliente sem length cap/CRLF check. Combinado com H8 = log injection. Fix: valid UUID/nanoid apenas.

### M22. CORS sem guarda AllowCredentials=true + AllowOrigin='*'
`middlewares/Poseidon.Middleware.CORS.pas:56-64` — Config inválida por spec passa silenciosamente. Fix: raise `EPoseidonException` na factory.

### M23. Parser Full `Trim + SameText` para chunked
`src/Poseidon.Net.HTTP1.Parser.pas:644` — Aloca `Trim(LValue)`. + inconsistência com Lightweight (aceita `chunked, gzip`). Fix: `_BytesEqualCI` sobre bytes.

### M24. Parser Full `hiConnection` chama `LowerCase(LValue)` 2×
`src/Poseidon.Net.HTTP1.Parser.pas:627-628` — 2 alocações da mesma string por request keep-alive. Fix: 1× em local var.

### M25. Epoll TCP_CORK on/off = 2 syscalls extras por response
`src/Poseidon.Net.IO.Epoll.pas:411-418` — 3 syscalls (cork-on + writev + cork-off) em vez de 1. Fix: usar `MSG_MORE` ou remover cork quando writev cabe.

### P1. `TRIOBackend.Destroy` não limpa FListenSocket/FCQs/FWorkerIOCPs/FWorkerOvls/FCQLocks

### P2. `TIOUringBackend.Destroy` não fecha `FListenSockets[]`
`src/Poseidon.Net.IO.IOUring.pas:446-480` — Safety-net cobre mmaps + ring fd mas não listen sockets. Exceção em setup pós-`CreateListenSocket` (linhas 534-536) vaza N fds. Fix: iterar `FListenSockets` com `_LinuxClose` em Destroy.

### P3. `TIOCPBackend._PostOneAccept` failure = slot permanentemente inativo
`src/Poseidon.Net.IO.IOCP.pas:281-317` — Falha de AcceptEx não `WSA_IO_PENDING` fecha socket + marca INVALID_SOCKET, mas não re-tenta post. Cenário: WSAENOBUFS transiente → slot never posts again → capacidade concurrent-accept degrada silenciosamente. Fix: logar erro + `PostQueuedCompletionStatus` com marca de retry.
